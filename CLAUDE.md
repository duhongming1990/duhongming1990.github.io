# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A GitHub Pages static site for comparing AI model outputs side-by-side. Each "topic" (e.g. 时序图, 向量数据库) has a folder containing standalone HTML pages generated by different AI models. The navigation page (`index.html`) loads them in an iframe grid for visual comparison.

## Architecture

```
index.html          — Navigation shell: navbar + iframe grid viewer
topics.js           — Data-only config: TOPICS array drives all navigation
<topic-folder>/     — One folder per topic, containing per-model HTML files
  ModelName.html    — Self-contained static page (no shared CSS/JS)
mkdocs.yml          — Legacy MkDocs config (not currently used for the comparison site)
```

**Key flow:** `index.html` loads `topics.js`, renders topic buttons from the `TOPICS` array, and on click builds a CSS grid of iframes pointing to the per-model HTML files.

## How to Add a New Topic

1. Create a folder (e.g. `new-topic-guide/`) with one HTML file per model.
2. Add an entry to the `TOPICS` array in `topics.js`.

That's it — no build step, no other files to touch.

## Development

No build tools or dependencies. Open `index.html` in a browser directly, or use any static server:

```sh
python3 -m http.server 8000
```

## Conventions

- Folder names use kebab-case (e.g. `ai-gateway-guide/`).
- HTML filenames match the model name exactly (e.g. `Claude-Opus-4.6.html`).
- Each model HTML page is fully self-contained (inline CSS/JS, no external deps).
- The grid layout auto-adapts: 2x2 for 4 pages, 3x2 for 5-6 pages, etc.
