🤔 什么是 AI Gateway？

🔬 深度分析：三大类别

AI 网关项目可分为三大类，满足不同场景需求

🎯

AI 专用网关（AI-Native）

代表：Portkey、LiteLLM、Helicone

从零为 AI/LLM 场景设计，内置语义缓存、Prompt 管理、Guardrails、成本追踪等 AI 专属功能。适合 AI 优先的团队。

🏗️

传统网关 + AI 扩展

代表：Kong、APISIX、Traefik

在成熟 API 网关上增加 AI 插件。适合已有网关基础设施、需要统一管理传统 API 和 AI API 的企业。

☁️

云原生 / 托管方案

代表：Envoy AI、Cloudflare、Gloo

深度集成 K8s/服务网格或云平台。适合重度云原生架构的团队，或希望零运维的小团队。

🏅 关键维度对比分析

1️⃣ 性能与延迟

AI Gateway 的核心指标是引入的额外延迟。对于实时对话类应用，每多 1ms 都很关键。

Portkey 9.5/10

Rust 核心 + Edge Workers，<1ms 额外延迟，支持流式传输零缓冲。生产环境 P99 表现最佳。

Envoy AI 9.0/10

C++ 核心引擎，数据面性能极强。但控制面 Go 代码有优化空间。

Kong AI 8.5/10

Nginx + Lua 架构成熟，性能稳定。AI 插件增加约 1-2ms。

LiteLLM 6.5/10

Python + FastAPI，单实例约 3-8ms 额外延迟，高并发下性能下降明显。

2️⃣ AI 专属功能丰富度

区别于传统 API 网关，AI 网关需要语义缓存、Guardrails、Prompt 管理、Token 计费等专属能力。

Portkey 9.5/10

功能最完整：语义缓存、Guardrails、Prompt 版本管理、A/B 测试、自动回退、成本分析、虚拟密钥。

LiteLLM 8.5/10

预算管理和团队管理出色，缓存和回退完善。缺少 Prompt 管理和高级 Guardrails。

Helicone 7.5/10

可观测性和分析无敌，但缺少路由、负载均衡等核心网关能力。

Kong AI 6.0/10

AI 功能主要通过插件实现，语义缓存和高级 AI 特性需要企业版。

3️⃣ 开发者体验 (DX)

Portkey 9.0/10

npm/pip 一键安装，OpenAI SDK 兼容，改一行 base_url 即可接入。提供多语言 SDK。

LiteLLM 8.5/10

Python 开发者首选，pip install 即用。YAML 配置清晰，Admin UI 实用。

Cloudflare 8.0/10

控制台点击创建，替换 URL 即可。零运维，但定制受限。

Envoy AI 5.0/10

需要 K8s 环境，CRD 配置复杂，学习曲线陡峭。不适合快速启动。

4️⃣ 生产就绪度 & 社区活跃度

Portkey 9.0/10

被大量生产环境使用，商业支持完善，文档质量高，每周迭代更新。Discord 社区活跃。

Kong AI 9.0/10

10 年历史，企业客户无数。AI 功能虽新，但平台可靠性极高。

LiteLLM 8.0/10

Star 增长最快，社区庞大。但 Python 在高负载下的稳定性有反馈问题。

Envoy AI 6.0/10

项目较新（2024），功能仍在快速开发中。CNCF 背书有潜力但尚不成熟。

🚀 Portkey AI Gateway 快速上手

3 种方式，5 分钟内启动你的 AI 网关

方式一：npx 一键启动（最快）

# 无需安装，直接运行
npx @portkey-ai/gateway

# 网关默认运行在 http://localhost:8787

方式二：Docker 部署

# 拉取并运行
docker run -d \
  --name portkey-gateway \
  -p 8787:8787 \
  portkeyai/gateway:latest

方式三：Node.js 项目集成

# 安装
npm install @portkey-ai/gateway

# 在代码中使用
import { Portkey } from 'portkey-ai';

const client = new Portkey({
  apiKey: "your-portkey-api-key",
  virtualKey: "your-openai-virtual-key"
});

// 使用方式与 OpenAI SDK 完全一致
const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "Hello!" }],
});

使用 OpenAI SDK 直接接入（零改动方案）

from openai import OpenAI

# 只需修改 base_url 和添加 header，其他代码完全不变！
client = OpenAI(
    api_key="your-openai-key",
    base_url="http://localhost:8787/v1",  # 👈 指向 Portkey 网关
    default_headers={
        "x-portkey-provider": "openai",
    }
)

# 之后的调用方式完全不变
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "你好"}]
)
