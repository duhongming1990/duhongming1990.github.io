⏱️ 5分钟直观了解：什么是时序图？

💡 一句话解释：
时序图（Sequence Diagram）就像是系统交互的“剧本”。它按照时间顺序（从上到下），展示了不同角色或对象之间是如何发送消息、互相配合完成一项任务的。

🧩 时序图的四大核心元素

🧍 参与者 (Actor)

发起操作的人或外部系统。通常画在最左边，用一个小人表示。

📦 对象 (Object)

系统内部的模块、服务器或数据库。通常用顶部的矩形框表示。

⬇️ 生命线 (Lifeline)

从对象往下垂的虚线，代表这个对象在一段时间内是存活/存在的。

✉️ 消息 (Message)

对象之间的连线（箭头）。实线代表发送请求，虚线代表返回结果。

🎬 实战演示：用户登录过程

下面是一个典型的“用户登录”时序图。请从上往下看，箭头代表消息的传递方向：

        sequenceDiagram
            autonumber
            actor User as 🧍 用户
            participant App as 📱 手机APP
            participant Server as 💻 后端服务器
            participant DB as 🗄️ 数据库

            User->>App: 1. 输入账号密码并点击登录
            activate App
            App->>Server: 2. 发送登录请求 (账号, 密码)
            activate Server
            Server->>DB: 3. 查询该账号信息
            activate DB
            DB-->>Server: 4. 返回账号数据
            deactivate DB
            
            alt 密码正确
                Server-->>App: 5. 返回登录成功 & Token
                App-->>User: 6. 跳转到首页
            else 密码错误
                Server-->>App: 5. 返回错误提示
                App-->>User: 6. 弹窗提示"密码错误"
            end
            deactivate Server
            deactivate App

📌 快速读图秘籍

  • 看顺序： 永远从最上面开始，往下看。越靠上的动作发生得越早。
  • 看箭头： ➔ 实线箭头是“我叫你做事”（请求），--> 虚线箭头是“我做完了告诉你”（返回响应）。
  • 看方块： 生命线上的竖长方形（激活期）表示这个对象正在“干活”（处理逻辑）。
  • 看分支： 图中的 alt 框代表“如果...否则...”（If-Else逻辑），比如密码正确和错误的不同走向。

现在你已经掌握了时序图的核心！它可以用来设计API、梳理业务逻辑、或者写技术文档。
