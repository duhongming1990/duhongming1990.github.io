Linux 文件系统速查：ZFS vs XFS vs ext4

ext4 稳定 兼容

🎯 使用场景
  • 通用首选：绝大多数Linux发行版默认。
  • 小文件多：源代码、配置文件、邮件服务。
  • 兼容性：需要与旧系统或Windows工具交互。
⚡ 安装配置
# 1. 创建文件系统
mkfs.ext4 /dev/sdb1

# 2. 挂载
mount /dev/sdb1 /mnt/data

# 3. 开机挂载 (/etc/fstab)
UUID=xxxx  /mnt/data  ext4  defaults  0 0
📦 典型软件/系统
  • Ubuntu / Debian / Mint
  • 传统 LAMP 服务器
  • 家用 Linux 工作站

XFS 高并发 大文件

🎯 使用场景
  • 大文件：视频剪辑、渲染、大型归档。
  • 高并发 I/O：大型数据库、虚拟化镜像。
  • 只能扩大：支持在线扩容，不能缩容。
⚡ 安装配置
# 1. 创建文件系统
mkfs.xfs /dev/sdb1

# 2. 挂载
mount /dev/sdb1 /mnt/data

# 3. 在线扩容 (如果LV变大)
xfs_growfs /mnt/data
📦 典型软件/系统
  • RHEL / CentOS / Rocky Linux
  • MySQL / MariaDB (大表)
  • GlusterFS (大文件存储)

ZFS 企业级 数据完整

🎯 使用场景
  • 数据安全：自带校验和，防止数据静默损坏。
  • 存储阵列：软RAID (RAID-Z)，NAS存储。
  • 快照备份：秒级快照，复制增量备份。
⚡ 安装配置
# 1. 安装 (Ubuntu为例)
apt install zfsutils-linux

# 2. 创建存储池 (RAIDZ)
zpool create mypool raidz sdb sdc sdd

# 3. 创建文件系统(数据集)
zfs create mypool/data

# 4. 开启压缩
zfs set compression=lz4 mypool/data
📦 典型软件/系统
  • TrueNAS / Proxmox VE
  • 高性能 NFS 服务器
  • Nextcloud 私有云
