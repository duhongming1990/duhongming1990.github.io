🎯 典型使用场景

🟢 ext4 适合场景

  • 系统根分区 / /boot — GRUB 原生支持最好
  • 桌面日常使用 — 稳定、零配置
  • 小型 VPS / 云服务器 — 内存不多时
  • 嵌入式 / 小存储 — 开销最小
  • 需要缩小分区 — 唯一支持的
  • Docker/容器 overlay2 — 默认后端

🔵 XFS 适合场景

  • 视频编辑 / 媒体存储 — 大文件顺序读写王者
  • 高并发数据库 — PostgreSQL/MySQL 数据目录
  • 企业级文件服务器 — RHEL 官方默认
  • 大规模并行写入 — 日志采集、监控数据
  • 虚拟机磁盘镜像存储 — KVM/QEMU 宿主
  • HPC / 科学计算大文件

🟠 ZFS 适合场景

  • NAS 存储 — TrueNAS 的灵魂
  • 数据备份服务器 — 快照 + 增量发送
  • Proxmox VE 虚拟化 — 快照/克隆虚拟机
  • 数据完整性要求极高 — 金融、医疗存档
  • 自建 RAID — 替代硬件 RAID 卡
  • 数据压缩需求 — 日志、文本文件可省 50%+

🔧 安装与配置

📦 安装（通常已内置）
# 内核原生支持，无需安装 # 工具包（一般已预装）: sudo apt install e2fsprogs # Debian/Ubuntu sudo dnf install e2fsprogs # RHEL/Fedora
🔨 创建与挂载
# 格式化 sudo mkfs.ext4 /dev/sdb1 # 带标签格式化 sudo mkfs.ext4 -L "mydata" /dev/sdb1 # 挂载 sudo mount /dev/sdb1 /mnt/data # 永久挂载 - 编辑 /etc/fstab: /dev/sdb1 /mnt/data ext4 defaults 0 2
⚙️ 优化调参
# 查看信息 sudo tune2fs -l /dev/sdb1 # 禁用日志（U盘/SD卡减少写入） sudo tune2fs -O ^has_journal /dev/sdb1 # 取消保留空间（非系统盘） sudo tune2fs -m 0 /dev/sdb1 # SSD 优化 fstab 选项: defaults,noatime,discard # 检查修复 sudo fsck.ext4 -f /dev/sdb1 # 在线扩容（先扩分区再扩FS） sudo resize2fs /dev/sdb1 # 缩小（需先卸载！） sudo resize2fs /dev/sdb1 50G
📦 安装
# RHEL系已预装，Debian/Ubuntu需装工具: sudo apt install xfsprogs # Debian/Ubuntu sudo dnf install xfsprogs # RHEL/Fedora(通常已有)
🔨 创建与挂载
# 格式化 sudo mkfs.xfs /dev/sdb1 # 强制格式化（覆盖已有FS） sudo mkfs.xfs -f /dev/sdb1 # 带标签 sudo mkfs.xfs -L "bigdata" /dev/sdb1 # 挂载 sudo mount /dev/sdb1 /mnt/data # 永久挂载 - /etc/fstab: /dev/sdb1 /mnt/data xfs defaults,noatime 0 0
⚙️ 优化调参
# 查看信息 xfs_info /mnt/data # 在线扩容（不支持缩小！） sudo xfs_growfs /mnt/data # 修复（需卸载） sudo xfs_repair /dev/sdb1 # 碎片整理（在线） sudo xfs_fsr /mnt/data # 冻结/解冻（一致性快照前用） sudo xfs_freeze -f /mnt/data sudo xfs_freeze -u /mnt/data # 大文件优化 fstab 选项: defaults,noatime,inode64,logbufs=8
⚠️ XFS 注意：不能缩小！创建前规划好大小。删除大量文件后空间不会立即释放给文件系统以外，需用 xfs_fsr 整理。
📦 安装
# === Ubuntu/Debian === sudo apt install zfsutils-linux # === RHEL/CentOS/Rocky === sudo dnf install https://zfsonlinux.org/epel/zfs-release-2-3.el9.noarch.rpm sudo dnf install zfs sudo modprobe zfs # === Proxmox VE（已内置）=== # 安装时可直接选 ZFS root # 验证 zfs version
🔨 创建存储池
# 单盘池 sudo zpool create mypool /dev/sdb # 镜像（RAID1） sudo zpool create mypool mirror /dev/sdb /dev/sdc # RAID-Z1（类RAID5，允许坏1盘） sudo zpool create mypool raidz /dev/sd{b,c,d} # RAID-Z2（类RAID6，允许坏2盘） sudo zpool create mypool raidz2 /dev/sd{b,c,d,e} # 添加缓存盘（SSD加速读取） sudo zpool add mypool cache /dev/nvme0n1 # 添加日志盘（SSD加速同步写入） sudo zpool add mypool log /dev/nvme0n2
📁 创建数据集 & 配置
# 创建数据集（类似子卷/子分区） sudo zfs create mypool/documents sudo zfs create mypool/videos sudo zfs create mypool/backups # 开启压缩（强烈推荐！） sudo zfs set compression=lz4 mypool # 或使用 zstd（更好压缩率） sudo zfs set compression=zstd mypool/backups # 设置配额 sudo zfs set quota=500G mypool/videos # 设置预留空间 sudo zfs set reservation=100G mypool/documents # 关闭 atime（性能优化） sudo zfs set atime=off mypool # 设置自动挂载点 sudo zfs set mountpoint=/data mypool
📸 快照与恢复（杀手特性）
# 创建快照（瞬间完成！） sudo zfs snapshot mypool/documents@2024-01-15 # 查看快照 sudo zfs list -t snapshot # 回滚到快照 sudo zfs rollback mypool/documents@2024-01-15 # 克隆快照为新数据集 sudo zfs clone mypool/documents@2024-01-15 mypool/docs-copy # 增量发送到远程（超强备份） sudo zfs send -i @snap1 mypool/data@snap2 | \ ssh remote zfs recv backup/data # 自动快照（安装工具） sudo apt install zfs-auto-snapshot
🔍 监控与维护
# 查看池状态 sudo zpool status sudo zpool list # 查看数据集 sudo zfs list # 查看压缩率 sudo zfs get compressratio mypool # 数据校验（定期运行！） sudo zpool scrub mypool # 替换故障盘 sudo zpool replace mypool /dev/sdb /dev/sdf # 导出/导入池（迁移硬盘组） sudo zpool export mypool sudo zpool import mypool
🧠 ZFS 内存建议：ZFS 使用 ARC 缓存，默认占用 50% 内存。每 TB 存储建议 1GB 内存。生产环境建议 ≥ 16GB。可通过 /etc/modprobe.d/zfs.conf 限制：options zfs zfs_arc_max=4294967296（4GB）

🏗️ 实战常见组合

🖥️ 桌面/开发机
/boot → ext4 (512MB, GRUB 兼容最好) / → ext4 (50GB, 系统根分区) /home → ext4 (剩余空间) swap → swap (内存等大或 swap file)
🏢 数据库服务器 (RHEL)
/boot → ext4 (1GB) / → xfs (50GB, RHEL默认) /var/lib/pgsql → xfs (独立SSD/NVMe) /var/log → xfs (独立盘, 避免日志撑爆系统)
💾 NAS / 备份服务器
/boot → ext4 (USB/独立小盘) / → ext4 或 ZFS (系统) tank → ZFS RAIDZ1 (3-4块数据盘) ├─ tank/share (Samba共享) ├─ tank/backup (备份, 压缩=zstd) ├─ tank/media (影音, 压缩=off) └─ tank/docker (容器数据) cache → SSD L2ARC (读缓存加速)
🖥️ Proxmox 虚拟化
/boot → ext4 (安装时自动) rpool → ZFS mirror (2×NVMe, 系统+虚拟机) ├─ rpool/ROOT (PVE 系统) ├─ rpool/data (VM 磁盘, zvol) └─ rpool/backup (vzdump 备份) # 快照虚拟机：秒级完成 # 克隆虚拟机：零拷贝
