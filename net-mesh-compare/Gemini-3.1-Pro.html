核心节点档案 (Protocols & Tools)

FRP 反向代理

专注于内网穿透的高性能反向代理应用，支持 TCP、UDP、HTTP、HTTPS 等协议。

  • 架构： 典型的 C/S 架构（客户端/服务端）。
  • 优势： 配置简单，按需暴露端口，适合建站或单点服务暴露。
  • 劣势： 流量需经过公网服务器中转，带宽受限于中转服务器；非 P2P 组网。

WireGuard 底层协议

被并入 Linux 内核的现代 VPN 协议，以极简的代码和极高的性能著称。

  • 架构： 去中心化的点对点（P2P）或星型拓扑。
  • 优势： 性能天花板，延迟极低，加密先进（ChaCha20）。
  • 劣势： 纯手动配置繁琐，无内置的 IP 分配和密钥分发管理机制，缺乏 NAT 穿透能力。

Headscale Mesh VPN

Tailscale 控制服务器的开源自建实现，底层基于 WireGuard 协议。

  • 架构： 控制面中心化（自建），数据面 P2P（WireGuard）。
  • 优势： 拥有 WireGuard 的性能，同时解决了密钥分发和 NAT 穿透（DERP）问题，全平台客户端支持极佳。
  • 劣势： 需要自己维护一台公网服务器作为控制节点。

ZeroTier SD-WAN

将整个地球视为一个交换机的二层虚拟局域网组网工具。

  • 架构： 根服务器（Planet）+ 卫星节点（Moon）+ 客户端（Leaf）。
  • 优势： 傻瓜式操作，支持二层网络（可广播），跨平台兼容性极强。
  • 劣势： 官方 Planet 节点在海外，国内直连打洞成功率受运营商影响较大（通常需自建 Moon 节点加速）。

EasyTier 去中心化组网

基于 Rust 编写的简单、安全、去中心化的内网穿透与 VPN 组网方案。

  • 架构： 完全去中心化，支持无公网 IP 组网。
  • 优势： 极简部署，自带公网共享节点，支持 WireGuard 协议，网络自愈能力强。
  • 劣势： 生态和知名度相对较新，第三方 GUI 客户端和周边工具仍在发展中。

决策指南 (Decision Protocol)

  • 如果你只有一台公网服务器，且只想把家里的 NAS 或 Web 服务暴露给朋友访问，不需要复杂的局域网互通：选择 FRP。
  • 如果你是网络极客，拥有固定的公网 IP，追求极致的吞吐量和极低的延迟，且不怕折腾路由表：选择 WireGuard。
  • 如果你不想买服务器，只想和朋友打局域网游戏，或者随时随地访问家里的电脑：选择 ZeroTier。
  • 如果你有多台设备（手机、电脑、NAS、软路由），希望它们像在同一个局域网内一样互相访问，且希望拥有美观的客户端和极高的安全性：选择 Headscale (Tailscale)。
  • 如果你喜欢 Rust 生态，追求极致的轻量化和去中心化，希望在没有公网 IP 的情况下也能快速拉起一个高性能的虚拟网络：选择 EasyTier。
