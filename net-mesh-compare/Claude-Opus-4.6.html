Overview

五大工具速览

不同设计哲学、不同使用场景，一次看懂它们各自在做什么

F
FRP
Fast Reverse Proxy
反向代理 · 内网穿透

由 fatedier 开发的高性能反向代理应用，专注于内网穿透。通过在公网服务器部署 frps，内网机器运行 frpc，实现将内网服务安全暴露到公网。支持 TCP / UDP / HTTP / HTTPS 等多种协议。

✓
端口映射，精准暴露指定服务
✓
支持自定义域名、HTTPS
✗
不支持设备间直接组网
✓
Dashboard 管理面板
H
Headscale
Self-hosted Tailscale
Mesh VPN · 自托管

Tailscale 控制平面的开源自托管实现。底层基于 WireGuard，提供零配置的 Mesh VPN 组网能力。设备通过 DERP 中继或直连打洞实现全互联，无需公网 IP，数据端到端加密。

✓
基于 WireGuard 的端到端加密
✓
自动 NAT 打洞 + DERP 中继
✓
ACL 访问控制、MagicDNS
◐
需部署控制服务器 + 客户端
Z
ZeroTier
Global Area Networking
虚拟局域网 · SD-WAN

创建虚拟以太网的 SD-WAN 平台，让分布在全球的设备像在同一局域网中一样通信。提供二层以太网仿真，支持组播广播，自带 Planet / Moon 中继架构，免费版支持 25 个设备。

✓
二层网络仿真，透明组网
✓
Web 控制台管理网络
◐
依赖官方 Planet 节点（可自建 Moon）
✓
跨平台支持极佳
W
WireGuard
Next-gen VPN Tunnel
VPN 隧道 · 内核级

极简、高速的现代 VPN 隧道协议，代码量仅约 4000 行，已合入 Linux 内核。采用最先进的密码学原语（Noise 协议框架、ChaCha20、Curve25519），性能远超 OpenVPN 和 IPSec。

✓
Linux 内核原生支持，极致性能
✓
密码学设计先进，攻击面小
✗
无内置 NAT 打洞 / 自动发现
◐
Peer 管理需手动配置
E
EasyTier
Decentralized Mesh VPN
去中心化 · P2P Mesh

Rust 编写的新一代去中心化 Mesh 组网工具。无需中心控制节点，节点间通过共享网络名+密钥自动发现并组网。支持 WireGuard 协议加密、多种打洞方式，内置公共节点辅助发现。

✓
真正去中心化，无单点故障
✓
零配置自动组网，一行命令启动
✓
WireGuard 加密 + 多协议支持
◐
项目较新，生态仍在发展中

从 C/S 到 Mesh，三种典型网络拓扑决定了根本差异

Client-Server 模式

FRP 采用经典 C/S 架构。所有流量都经过中心公网服务器转发，客户端不直接通信。简单可靠，但服务器成为性能瓶颈和单点故障。

Node A
Node B

控制面 + 数据直连

Headscale / ZeroTier 分离控制面和数据面。控制服务器仅负责身份认证和节点发现，数据尽量点对点直连（P2P），打洞失败时回退中继。

Peer
Peer

Full Mesh / 去中心化

WireGuard 纯 Peer-to-Peer（需手动配置）。EasyTier 在此基础上加入自动发现和打洞，实现真正的去中心化 Mesh，无需任何中心服务。

Technical Details

技术细节深挖

安全模型、协议栈和网络特性的关键差异

🔐 安全与加密对比

  • FRP：TLS 加密传输通道，Token 认证客户端。安全性取决于 frps 的部署配置，数据经服务器中转意味着服务器可见明文
  • Headscale / Tailscale：基于 WireGuard 的端到端加密，控制面仅交换公钥。即使 DERP 中继转发，中继节点也无法解密数据
  • ZeroTier：自研加密方案，每个网络有唯一 256 位网络 ID，节点间使用 Curve25519 + Salsa20 加密
  • WireGuard：密码学界高度评价。Noise 协议框架 + ChaCha20-Poly1305 + BLAKE2s，代码量小使审计成为可能
  • EasyTier：复用 WireGuard 协议加密通道，继承其安全性优势。通过共享密钥（network secret）实现组网认证

🌐 NAT 穿透能力

  • FRP：不涉及穿透，要求服务端必须有公网 IP。这是最大的限制，但也让其行为最可预测
  • Headscale：使用 STUN 探测 NAT 类型，通过协调服务器交换候选地址实现打洞；失败时回退 DERP 中继（可自建）
  • ZeroTier：多层穿透策略，包括 UDP 打洞、UPnP/NAT-PMP、端口预测等。Planet 根服务器提供最后的中继保障
  • WireGuard：协议层面不处理 NAT 穿透。需要手动指定 Endpoint 或借助 persistent-keepalive 维持映射
  • EasyTier：内置 UDP/TCP 打洞、端口映射检测等多种策略。使用公共发现节点辅助建立直连

⚡ 性能与吞吐

  • WireGuard 凭借内核态实现一骑绝尘，在合适硬件上可达接近线速（10Gbps+），延迟最低
  • Headscale/Tailscale 直连时继承 WireGuard 性能；经 DERP 中继会受限于中继服务器带宽
  • ZeroTier 用户态实现，直连时性能良好（通常可达数百 Mbps），但 Planet 中继带宽有限
  • FRP 所有流量经中转服务器，吞吐量完全取决于服务器带宽。使用 XTCP 可实现 P2P 但成功率有限
  • EasyTier Rust 用户态实现，性能持续优化中。直连时可达数百 Mbps，比内核态 WG 略低但足够实用

🛠 运维与管理

  • FRP：TOML 配置文件驱动，自带 Web Dashboard。适合运维人员手动管理少量穿透规则
  • Headscale：提供 CLI + API + 第三方 UI（如 headscale-ui）。ACL 策略强大但学习曲线存在
  • ZeroTier：官方 Web 控制台体验最佳，可视化管理网络成员、路由规则。企业版提供更多管控功能
  • WireGuard：纯命令行 wg / wg-quick 管理。无内置 UI，社区有诸多 GUI 封装
  • EasyTier：CLI 启动 + Web 控制台。配置极简，但高级功能（如路由策略）还在完善中

Pros & Cons

优势与局限

没有完美的工具，只有最适合的选择

F

FRP

  • ✦部署简单直观，概念清晰。对于暴露单个服务（如 Web、SSH）到公网非常高效
  • ✦支持自定义域名绑定、HTTP 反向代理，适合 Web 开发者和小型服务发布
  • ✦社区成熟，GitHub 88k+ Star，文档完善，遇到问题易找解答
  • ✧必须拥有公网 IP 的服务器，存在服务器租用成本
  • ✧所有流量走中转，带宽和延迟受服务器限制，不适合大流量场景
  • ✧不具备组网能力，无法实现多设备互联互通
H

Headscale

  • ✦完全自托管，数据主权在自己手中。兼容 Tailscale 全平台客户端生态
  • ✦端到端 WireGuard 加密，直连性能极高。ACL 权限控制精细灵活
  • ✦MagicDNS 自动解析、子网路由、Exit Node 等企业级功能完备
  • ✧需要搭建和维护控制服务器，有一定的运维门槛
  • ✧控制面宕机时新设备无法加入（已连接设备仍可通信）
  • ✧相比 Tailscale 官方版，部分高级功能（如 Funnel）不可用
Z

ZeroTier

  • ✦开箱即用体验最佳，注册 → 创建网络 → 加入，三步完成组网
  • ✦L2 以太网仿真支持广播/组播，适合游戏联机、SMB 共享等场景
  • ✦Web 控制台直观易用，跨平台覆盖最广（含 iOS/Android/NAS）
  • ✧免费版限 25 节点。官方 Planet 节点在中国大陆延迟较高
  • ✧BSL 许可证非真正开源，自建 Moon 节点可改善但无法完全脱离官方基础设施
  • ✧自研加密方案未经密码学社区大规模审计
W

WireGuard

  • ✦性能之王。内核态运行，加密开销极小，延迟和吞吐均为顶级水平
  • ✦代码量仅 ~4000 行，安全性经过密码学专家严格审计，是 VPN 安全标杆
  • ✦已合入 Linux 5.6+ 内核，主流发行版原生支持，长期稳定有保障
  • ✧不具备自动发现、NAT 穿透能力，需手动管理每个 Peer 的密钥和 Endpoint
  • ✧节点规模增长后配置管理复杂度呈 O(n²) 增长（Full Mesh）
  • ✧是"基础设施"而非"产品"，需要上层工具（Tailscale/Headscale/EasyTier）包装才好用
E

EasyTier

  • ✦真正的去中心化——无需任何服务器，一行命令即可组建 Mesh 网络
  • ✦Rust 编写，内存安全有保障。同时支持 WireGuard 和自有协议
  • ✦支持子网代理、多协议监听（TCP/UDP/WS/WSS），灵活适配复杂网络环境
  • ✧项目较新（2024），社区规模和文档完善度仍在成长期
  • ✧用户态实现，极限性能不及内核态 WireGuard
  • ✧公共发现节点稳定性取决于社区维护，生产环境建议自建

Quick Decision

快速决策指南

回答几个关键问题，快速找到你的最优解

🤔 你需要的是什么？

  • 只是把内网一个端口映射出去？ → 直接用 FRP，最简单直接
  • 需要多台设备互相访问？ → 这是"组网"需求，往下看
  • 追求极致性能和安全？ → WireGuard 是底层最优解
  • 想要 WireGuard 的体验但不想手动配置？ → Headscale
  • 不想搭建任何服务器？ → EasyTier 或 ZeroTier

💡 关键考量因素

  • 有无公网 IP / VPS？ 有 → FRP / WireGuard 可选；无 → 选 ZeroTier / EasyTier / Headscale
  • 在中国大陆使用？ ZeroTier 官方节点较慢需自建 Moon；Headscale 需自建 DERP；EasyTier 公共节点在国内可用
  • 设备数量？ <5 台 → 任何方案均可；5-25 台 → ZeroTier 免费版够用；25+ → Headscale 或 WireGuard
  • 运维能力？ 弱 → ZeroTier / EasyTier；中等 → FRP / Headscale；强 → WireGuard 手动组网
  • 数据敏感度？ 极高 → WireGuard / Headscale（端到端加密 + 自托管）；一般 → 任意

总结：一句话定位每个工具

它们不是相互替代的关系，而是解决不同层面问题的互补方案。WireGuard 是地基，Headscale 是在其上盖的智慧大楼，ZeroTier 和 EasyTier 是开箱即住的公寓，FRP 是一扇通往外界的精准之门。
