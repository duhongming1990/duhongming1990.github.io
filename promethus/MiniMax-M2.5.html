⚡Pull Model (拉取模式)

Prometheus 默认采用 主动拉取 的方式采集数据，而不是由客户端推送。

  • 服务端周期性访问 Target 的 HTTP 端点 (/metrics)。
  • 优势：无侵入性，便于统一管理
  • 组件：Node Exporter, Blackbox Exporter

0 ↗Counter (计数器)

一个 只增不减 的计数器。

  • 用途：请求次数、错误总数、流量字节数。
  • 分析：必须配合 rate() 或 increase() 使用。
  • 示例：rate(http_requests_total[5m])

↕Gauge (仪表盘)

表示一个 可任意变化 的数值。

  • 用途：当前内存使用量、在线人数、CPU温度。
  • 分析：直接查看当前值，或计算差值。
  • 操作：delta(), holt_winters()

📊Histogram (直方图)

将数据统计分布划分到预设的桶 (Buckets) 中。

  • 用途：请求延迟 (< 0.1s, < 0.5s, < 1s)。
  • 优势：服务端计算，适合高基数数据。
  • 自动生成：_bucket, _sum, _count

%Summary (摘要)

直接在客户端计算 分位数 (Quantiles)。

  • 用途：请求延迟的中位数 (0.5)、P99 (0.99)。
  • 缺点：不能聚合 (高基数问题)。
  • 选择：Histogram 用于可聚合场景，Summary 用于需要精确分位数的场景。

∑PromQL (查询语言)

Prometheus 的核心强大之处，用于对时序数据进行聚合和运算。

  • 瞬时向量 (Instant Vector): 返回某一时刻的一组时间序列。
  • 区间向量 (Range Vector): 返回某一时间段内的数据。
  • 常用函数：rate(), irate(), avg(), topk()

🔍Service Discovery (服务发现)

在动态环境中（如 Kubernetes）自动发现监控目标。

  • 静态配置：适用于 IP 固定的场景。
  • 动态发现：支持 Kubernetes, Consul, DNS 等。
  • relabel_configs: 重新标记 (Relabeling) 是最核心的过滤逻辑。
