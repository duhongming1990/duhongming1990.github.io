Prometheus 核心概念

云原生时代的监控标准，掌握这些核心概念，轻松驾驭系统可观测性。

数据模型 (Data Model)

Prometheus 将所有数据存储为时间序列 (Time Series)。每个时间序列由指标名称和一组键值对唯一标识。

  • Metric Name: 描述系统特征（如 http_requests_total）。
  • Labels (标签): 键值对，提供多维度的查询能力（如 method="GET"）。
  • Samples (样本): 包含一个 float64 值和一个毫秒级时间戳。

四大指标类型 (Metric Types)

客户端库提供了四种核心的指标类型，用于不同场景的数据采集：

  • Counter (计数器): 只增不减的累加指标（如请求总数、错误总数）。
  • Gauge (仪表盘): 可增可减的瞬时值（如当前内存使用量、并发数）。
  • Histogram (直方图): 对观察结果进行采样并分桶统计（如请求延迟分布）。
  • Summary (摘要): 类似于直方图，但直接在客户端计算分位数（如 P99 延迟）。

PromQL (查询语言)

Prometheus 内置的图灵完备查询语言，专为时间序列数据设计，功能极其强大。

  • 实时聚合: 支持 sum, avg, min, max 等聚合操作。
  • 向量计算: 支持即时向量（Instant Vector）和区间向量（Range Vector）的数学运算。
  • 灵活过滤: 通过标签选择器精准定位数据（如 {status="500"}）。

Pull 模型 (拉取机制)

与传统的 Push（推送）模型不同，Prometheus 默认采用 Pull（拉取） 方式获取数据。

  • 主动抓取: Server 定期通过 HTTP 请求从目标（Targets）的 /metrics 接口拉取数据。
  • 解耦与安全: 降低了客户端的复杂性，防止海量推送压垮监控服务端。
  • Pushgateway: 针对短暂运行的批处理任务，提供中间网关接收推送数据。

服务发现 (Service Discovery)

在动态的云原生环境中，监控目标 IP 随时变化，静态配置无法满足需求。

  • 动态感知: 自动发现需要监控的目标实例。
  • 原生集成: 完美支持 Kubernetes, Consul, EC2, Azure 等多种环境。
  • Relabeling (重标签): 在抓取前动态重写、过滤或丢弃目标的标签，极具灵活性。

核心生态组件 (Ecosystem)

Prometheus 不是一个孤立的工具，而是一个完整的监控生态系统。

  • Prometheus Server: 核心大脑，负责抓取、存储数据并提供查询接口。
  • Exporters: 各种数据采集器（如 Node Exporter, MySQL Exporter）。
  • Alertmanager: 负责处理告警，支持去重、分组、路由（邮件、钉钉、Webhook）。
  • Grafana: 黄金搭档，用于将 PromQL 查询结果进行炫酷的可视化展示。
